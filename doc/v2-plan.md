# V2 版本规划

## 主题

V2 的主题是“可控迁移能力”。V1 已经完成扫描、安全清理、日志回滚和只读迁移预检查；V2 开始把迁移能力做成可执行、可回滚的完整流程。

## V2 目标

- 用户默认目录迁移执行。
- 迁移回滚。
- 开发工具缓存迁移方案落地。
- 环境变量迁移。
- Junction 迁移预案。
- 更完整的任务进度和取消能力。

## V2.0：用户目录迁移执行

范围：

- 选择目标根目录。
- 创建目标目录。
- 移动用户目录内容。
- 修改 `User Shell Folders`。
- 写入迁移日志。
- 支持迁移回滚。

安全边界：

- 必须先通过预检查。
- 必须二次确认。
- 必须记录原注册表值。
- 必须记录原路径和目标路径。
- 失败时不能删除原目录。

当前状态：

- 已接入迁移计划命令。
- 已接入迁移执行命令。
- 已接入迁移日志记录。
- 已接入迁移回滚命令。
- 已在 Migration 页面展示计划、执行和结果。
- 已在 Logs 页面接入迁移回滚入口。

## V2.1：开发工具迁移

范围：

- Cargo / Rustup 环境变量迁移。
- Android 用户目录环境变量迁移。
- Bun 目录 Junction 迁移预案。
- 通用缓存仅清理，不迁移。
- AWS 等凭据目录继续只读提示。

安全边界：

- 默认不迁移凭据目录。
- Junction 必须可检测、可回滚。
- 环境变量修改必须记录原值。

## V2.2：任务体验

范围：

- 长任务进度事件。
- 扫描/迁移取消能力。
- 更明确的任务队列状态。
- 失败项导出。

## V2 暂不做

- 自动迁移 `Program Files`。
- 自动迁移 Docker / WSL / 虚拟机数据。
- 注册表深度清理。
- 永久删除隔离区文件。
