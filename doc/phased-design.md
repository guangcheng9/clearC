# 分阶段设计

本文件只保留阶段设计方法，具体版本范围以 [版本路线图](./roadmap.md) 为准，具体执行任务以 [执行文档](./execution-plan.md) 为准。

## 阶段划分原则

1. 先只读扫描，再执行清理。
2. 先低风险清理，再做迁移。
3. 先官方配置迁移，再考虑 Junction。
4. 先用户目录和标准开发缓存，再处理高风险专业场景。
5. 先本地能力稳定，再做规则更新和产品化发布。

## 功能推进顺序

| 阶段 | 类型 | 说明 |
|---|---|---|
| 阶段 0 | 工程基础 | 项目骨架、页面框架、规则系统、安全边界 |
| 阶段 1 | 只读扫描 | 磁盘空间、目录大小、规则命中、风险分类 |
| 阶段 2 | 安全清理 | 临时文件、回收站、缓存、日志 |
| 阶段 3 | 用户迁移 | 下载、文档、图片、视频、桌面等 Shell Folder 迁移 |
| 阶段 4 | 开发空间 | AI 工具、开发工具、SDK、包管理器缓存治理 |
| 阶段 5 | 深度分析 | 大文件、重复文件、扫描历史、智能建议 |
| 阶段 6 | 产品发布 | 安装包、便携版、规则更新、崩溃日志 |

## 阶段准入条件

- 核心流程可运行。
- 失败状态可展示。
- 不会因为权限不足崩溃。
- 有最小日志记录。
- 修改系统状态的功能有确认步骤。
