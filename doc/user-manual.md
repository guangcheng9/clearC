# 用户操作手册

本文档面向 ClearC 使用者，说明当前已经实现的功能、操作步骤、正确结果和异常结果。后续新增功能、修改按钮流程、调整结果展示或改变安全边界时，必须同步更新本文档。

## 使用前须知

- ClearC 是 Windows 系统盘治理工具，核心原则是先扫描、再确认、可回滚。
- 当前不会自动处理 `Program Files`、`.aws`、Docker、WSL、虚拟机和授权凭据目录。
- 涉及真实清理或迁移的操作必须由用户点击触发。
- 用户目录迁移会修改 Windows `User Shell Folders`，执行前请确认目标盘稳定可用。
- 当前用户目录迁移和回滚代码已完成，但仍待真实用户目录手动验收。正式迁移前建议先备份重要文件。

## 页面总览

| 页面 | 当前功能 | 是否会修改文件或系统配置 |
|---|---|---|
| 首页 | 查看 Native Core 状态、规则数量、系统盘概览 | 否 |
| 清理 | 扫描清理项、生成清理预览、对用户临时目录执行隔离清理 | 隔离清理会移动 `%TEMP%` 直接子项 |
| 迁移 | 检测用户默认目录、预检查、生成迁移计划、执行用户目录迁移 | 执行迁移会移动文件并修改注册表 |
| 开发工具 | 扫描 AI 工具、SDK、包管理器缓存和配置目录 | 否 |
| 分析 | 展示规则扫描结果和系统盘空间状态 | 否 |
| 日志 | 查看操作记录、回滚隔离清理和用户目录迁移 | 回滚会移动文件，迁移回滚会恢复注册表 |

## 首页

### 功能

- 显示应用基础状态。
- 显示 Native Core 是否可用。
- 展示系统盘容量概览。
- 展示清理、迁移、开发工具规则数量。

### 操作步骤

1. 打开应用。
2. 进入左侧导航的“首页”。
3. 查看顶部 Native Core 状态和页面中的容量/规则信息。

### 正确结果

- Native Core 显示 `Ready`。
- 系统盘容量信息可以正常展示。
- 规则数量不为异常空值。

### 错误或异常结果

- Native Core 显示 `Pending`：说明 Rust 后端状态读取失败或应用尚未就绪。
- 容量为空或读取失败：可能是系统盘信息读取失败，需要重新打开应用或检查权限。

## 清理

### 功能

- 查看清理规则。
- 扫描安全清理项。
- 生成清理预览。
- 当前真实清理仅支持 `%TEMP%` 用户临时目录的隔离移动。
- 隔离清理不会永久删除文件。

### 操作步骤

1. 进入“清理”页面。
2. 点击扫描或预览相关按钮。
3. 等待 loading 文案结束。
4. 查看预计释放空间、文件数量、跳过项和风险等级。
5. 如果页面显示可执行的隔离清理入口，确认后执行。
6. 执行后到“日志”页面查看操作记录。

### 正确结果

- 扫描期间按钮进入 loading 状态，页面不会卡死。
- 预览结果显示预计释放空间和规则明细。
- 隔离清理后显示移动数量、跳过数量和失败项。
- 日志页面出现 `cleanup-quarantine` 类型记录，并在可回滚时显示回滚按钮。

### 错误或异常结果

- 按钮不可点击：通常是前置条件未满足，例如还未扫描或该规则不允许执行。
- 失败项中出现权限不足、文件占用：表示文件被系统或其他软件使用，属于可接受的跳过结果。
- 原路径已存在导致回滚跳过：说明目标恢复位置已有同名文件，应用不会覆盖。

### 注意事项

- 当前不会永久删除文件。
- 当前不会清空回收站。
- 当前只开放用户临时目录的隔离清理，不处理系统关键目录。

## 迁移

### 功能

- 检测 Windows 默认用户目录：桌面、下载、文档、图片、视频、音乐。
- 读取每个目录当前 `User Shell Folders` 配置。
- 判断目录是否仍在系统盘。
- 对目标根目录执行预检查。
- 生成迁移计划。
- 在用户确认后执行迁移。
- 执行结果写入日志，并可在“日志”页面回滚。

### 操作步骤

1. 进入“迁移”页面。
2. 在“目标根目录”输入目标盘路径，例如 `D:\ClearC-UserFolders`。
3. 在“当前用户目录”中选择一个目录，点击“预检查”。
4. 预检查通过后，点击“生成计划”。
5. 在“迁移计划与结果”中查看源路径和目标路径。
6. 在确认短语输入框中输入 `MIGRATE_USER_FOLDER`。
7. “执行迁移”按钮可用后点击执行。
8. 在弹窗中再次确认源路径和目标路径。
9. 执行完成后查看移动项、跳过项和失败项。
10. 到“日志”页面查看 `user-folder-migration` 记录。

### 正确结果

- 预检查显示“通过”。
- 目标路径不是已有目录。
- 目标盘存在且剩余空间足够。
- 生成计划后能看到源路径、目标路径和 ready 状态。
- 输入完整 `MIGRATE_USER_FOLDER` 后，“执行迁移”按钮变为可用。
- 执行完成后状态为 `completed` 或 `completed-with-skips`。
- 日志中记录原注册表值、新注册表值、移动项和失败项。

### 错误或阻塞结果

- `源目录不存在`：当前 Windows 默认目录路径不存在，不能迁移。
- `目标目录已存在`：目标目录已有内容，应用不会覆盖，需要更换目标根目录或手动处理目标目录。
- `目标盘不存在或无法读取容量`：目标盘符不可用，检查磁盘是否挂载。
- `目标盘剩余空间不足`：目标盘可用空间小于源目录占用。
- `目标路径仍在系统盘`：可以继续作为警告展示，但不符合迁出系统盘目标。
- `completed-with-registry-failure`：文件移动有结果，但注册表写入失败，需要查看日志并谨慎处理。
- 失败项中出现文件占用或权限不足：对应文件未移动，应用不会强制处理。

### 注意事项

- 迁移只移动源目录的直接子项，不删除源目录本身。
- 目标路径已存在时不会覆盖。
- 跨盘迁移会使用受控复制、大小校验、删除源文件流程。
- 注册表更新后，资源管理器显示可能延迟，必要时重新登录或重启资源管理器。
- 当前未在本机真实用户目录上做自动测试，正式迁移前建议备份重要文件。

## 开发工具

### 功能

- 扫描开发工具、AI 工具、SDK、包管理器缓存和配置目录。
- 当前覆盖 Android、Cargo、Bun、通用缓存、Chromium snapshots、Claude、CodeBuddy、Apifox、AWS 等规则。
- 展示路径是否存在、占用大小、文件数量、风险等级和推荐动作。

### 操作步骤

1. 进入“开发工具”页面。
2. 点击扫描按钮。
3. 等待 loading 文案结束。
4. 查看每个工具目录的占用和建议。

### 正确结果

- 扫描完成后显示各规则的路径、大小和状态。
- 高风险目录只展示，不提供自动处理入口。
- `.aws` 等凭据目录只读展示。

### 错误或异常结果

- 目录显示缺失：说明当前用户环境没有该工具目录。
- 访问失败或跳过项增加：说明存在权限限制或文件占用。
- 未显示迁移按钮：当前阶段开发工具只读扫描，不支持自动迁移。

### 注意事项

- 当前不会清理开发工具目录。
- 当前不会移动开发工具目录。
- 当前不会修改环境变量。
- 当前不会创建 Junction。

## 分析

### 功能

- 展示系统盘概览。
- 展示规则路径扫描结果。
- 帮助判断空间占用来源。

### 操作步骤

1. 进入“分析”页面。
2. 点击扫描相关按钮。
3. 等待扫描完成。
4. 查看各路径的大小、文件数量、目录数量和跳过项。

### 正确结果

- 扫描期间有 loading 状态。
- 扫描完成后显示每条规则的统计结果。
- 跳过项不会导致扫描整体失败。

### 错误或异常结果

- 路径不存在：说明规则路径在当前电脑上不存在。
- 权限不足：说明该目录受系统保护或当前用户无权访问。
- 扫描耗时较长：大目录或大量小文件会增加耗时。

## 日志与回滚

### 功能

- 查看清理和迁移操作记录。
- 查看计划草稿、可回滚数量和最近操作。
- 回滚隔离清理。
- 回滚用户目录迁移。

### 操作步骤：回滚隔离清理

1. 进入“日志”页面。
2. 找到 `cleanup-quarantine` 类型记录。
3. 点击“回滚清理”。
4. 在确认弹窗中确认。
5. 查看恢复数量、跳过数量和失败项。

### 操作步骤：回滚用户目录迁移

1. 进入“日志”页面。
2. 找到 `user-folder-migration` 类型记录。
3. 点击“回滚迁移”。
4. 在确认弹窗中确认。
5. 查看恢复数量、跳过数量、注册表恢复状态和失败项。

### 正确结果

- 可回滚记录显示回滚按钮。
- 回滚期间显示 loading 文案。
- 回滚成功后写入新的不可回滚日志。
- 同一个迁移或隔离清理不能重复回滚。

### 错误或异常结果

- `operation is not rollbackable`：该记录不是可回滚操作。
- `migration already has a rollback record`：该迁移已经回滚过，不能重复回滚。
- 原路径已存在：应用不会覆盖已有文件，失败项会记录。
- 目标文件缺失：待恢复文件不存在，失败项会记录。
- `rolled-back-with-registry-failure`：文件回滚有结果，但注册表恢复失败，需要手动检查 Windows 默认目录配置。

## 白屏或页面异常

### 功能

- 应用已加入前端错误兜底。
- 页面渲染异常时会显示“页面渲染异常”面板，而不是整页白屏。

### 正确结果

- 普通操作不会出现白屏。
- 如果出现错误面板，可以看到错误信息，并点击“重新显示页面”尝试恢复。

### 错误或异常结果

- 如果仍然整页白屏，说明异常发生在错误兜底之前或 WebView 本身崩溃。
- 此时需要记录操作步骤、最后点击的按钮、输入内容，并查看开发日志。

## 判断操作是否正确

| 操作类型 | 正确表现 | 错误或需要关注 |
|---|---|---|
| 扫描 | 有 loading，完成后显示统计结果 | 长时间无响应、无错误提示、页面白屏 |
| 预检查 | 显示通过或明确阻塞原因 | 阻塞原因为空但按钮不可用 |
| 生成计划 | 显示源路径、目标路径、ready/blocked | 计划路径不符合预期 |
| 执行迁移 | 要求确认短语和弹窗确认 | 未确认就执行、目标路径已存在仍执行 |
| 清理 | 只隔离移动，不永久删除 | 删除了非临时目录或系统目录 |
| 回滚 | 写入新日志，重复回滚被拒绝 | 覆盖已有文件、重复回滚成功 |
