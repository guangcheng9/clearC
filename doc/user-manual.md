# 用户操作手册

本文档面向 ClearC 使用者，说明当前已经实现的功能、操作步骤、正确结果和异常结果。后续新增功能、修改按钮流程、调整结果展示或改变安全边界时，必须同步更新本文档。

## 使用前须知

- ClearC 是 Windows 系统盘治理工具，核心原则是先扫描、再确认、可回滚。
- 当前不会自动处理 `Program Files`、`.aws`、Docker、WSL、虚拟机和授权凭据目录。
- 涉及真实清理或迁移的操作必须由用户点击触发。
- 用户目录迁移会修改 Windows `User Shell Folders`，执行前请确认目标盘稳定可用。
- 当前用户目录迁移/回滚、开发工具环境变量迁移/回滚、开发工具 Junction 执行/回滚代码已完成，但仍待真实手动验收记录。正式迁移前建议先备份重要文件，并优先在测试目录或测试账号中验证。

## 页面总览

| 页面 | 当前功能 | 是否会修改文件或系统配置 |
|---|---|---|
| 首页 | 查看 Native Core 状态、规则数量、系统盘概览 | 否 |
| 清理 | 扫描清理项、生成清理预览、对用户临时目录执行隔离清理 | 隔离清理会移动 `%TEMP%` 直接子项 |
| 迁移 | 检测用户默认目录、预检查、生成迁移计划、执行用户目录迁移 | 执行迁移会移动文件并修改注册表 |
| 开发工具 | 扫描 AI 工具、SDK、包管理器缓存和配置目录 | 否 |
| 开发工具环境变量迁移 | 为 Android、Cargo、Rustup 设置用户级环境变量 | 会修改用户级环境变量 |
| 开发工具 Junction 预案 | 为适合 Junction 的工具生成迁移评估计划 | 否 |
| 分析 | 展示规则扫描结果和系统盘空间状态 | 否 |
| 日志 | 查看任务状态、导出失败项、回滚隔离清理和迁移操作 | 回滚会移动文件或恢复配置 |

## 首页

### 功能

- 显示应用基础状态。
- 显示 Native Core 是否可用。
- 展示系统盘容量概览。
- 展示清理、迁移、开发工具规则数量。

### 操作步骤

1. 打开应用。
2. 进入左侧导航的“首页”。
3. 查看顶部 Native Core 状态和页面中的容量/规则信息。

### 正确结果

- Native Core 显示 `Ready`。
- 系统盘容量信息可以正常展示。
- 规则数量不为异常空值。

### 错误或异常结果

- Native Core 显示 `Pending`：说明 Rust 后端状态读取失败或应用尚未就绪。
- 容量为空或读取失败：可能是系统盘信息读取失败，需要重新打开应用或检查权限。

## 清理

### 功能

- 查看清理规则。
- 扫描安全清理项。
- 生成清理预览。
- 当前真实清理仅支持 `%TEMP%` 用户临时目录的隔离移动。
- 隔离清理不会永久删除文件。

### 操作步骤

1. 进入“清理”页面。
2. 点击扫描或预览相关按钮。
3. 等待 loading 文案结束。
4. 查看预计释放空间、文件数量、跳过项和风险等级。
5. 如果页面显示可执行的隔离清理入口，确认后执行。
6. 执行后到“日志”页面查看操作记录。

### 正确结果

- 扫描期间按钮进入 loading 状态，页面不会卡死。
- 生成清理预览和隔离清理期间会显示进度条和当前处理内容。
- 生成清理预览期间会显示当前扫描路径、文件数、目录数、跳过数和扫描字节数。
- 隔离清理期间会显示已处理项、已移动项、跳过项、失败项和已处理字节数。
- 隔离清理进入复制 fallback 时，会显示当前文件路径、当前文件已复制字节和当前文件总字节。
- 生成清理预览和隔离清理期间可以点击“取消当前任务”，任务会在下一次检查点停止。
- 预览结果显示预计释放空间和规则明细。
- 隔离清理后显示移动数量、跳过数量和失败项。
- 日志页面出现 `cleanup-quarantine` 类型记录，并在可回滚时显示回滚按钮。

### 错误或异常结果

- 按钮不可点击：通常是前置条件未满足，例如还未扫描或该规则不允许执行。
- 失败项中出现权限不足、文件占用：表示文件被系统或其他软件使用，属于可接受的跳过结果。
- 原路径已存在导致回滚跳过：说明目标恢复位置已有同名文件，应用不会覆盖。

### 注意事项

- 当前不会永久删除文件。
- 当前不会清空回收站。
- 当前只开放用户临时目录的隔离清理，不处理系统关键目录。
- 隔离清理优先使用快速移动；只有快速移动失败时才进入分块复制 fallback。

## 迁移

### 功能

- 检测 Windows 默认用户目录：桌面、下载、文档、图片、视频、音乐。
- 读取每个目录当前 `User Shell Folders` 配置。
- 判断目录是否仍在系统盘。
- 对目标根目录执行预检查。
- 生成迁移计划。
- 在用户确认后执行迁移。
- 执行结果写入日志，并可在“日志”页面回滚。

### 操作步骤

1. 进入“迁移”页面。
2. 在“目标根目录”输入目标盘路径，例如 `D:\ClearC-UserFolders`。
3. 在“当前用户目录”中选择一个目录，点击“预检查”。
4. 预检查通过后，点击“生成计划”。
5. 在“迁移计划与结果”中查看源路径和目标路径。
6. 在确认短语输入框中输入 `MIGRATE_USER_FOLDER`。
7. “执行迁移”按钮可用后点击执行。
8. 在弹窗中再次确认源路径和目标路径。
9. 执行完成后查看移动项、跳过项和失败项。
10. 到“日志”页面查看 `user-folder-migration` 记录。

### 正确结果

- 预检查显示“通过”。
- 目标路径不是已有目录。
- 目标盘存在且剩余空间足够。
- 生成计划后能看到源路径、目标路径和 ready 状态。
- 迁移预检查、生成计划和执行迁移期间会显示进度条和当前处理内容。
- 用户目录迁移执行期间会显示已处理项、已移动项、跳过项、失败项和已处理字节数。
- 跨盘复制 fallback 期间会显示当前文件路径、当前文件已复制字节和当前文件总字节。
- 用户目录迁移执行期间可以点击“取消当前任务”，普通条目会在下一次文件处理检查点停止；跨盘复制 fallback 会在分块复制检查点停止，并清理部分复制出的目标文件。
- 输入完整 `MIGRATE_USER_FOLDER` 后，“执行迁移”按钮变为可用。
- 执行完成后状态为 `completed` 或 `completed-with-skips`。
- 日志中记录原注册表值、新注册表值、移动项和失败项。

### 错误或阻塞结果

- `源目录不存在`：当前 Windows 默认目录路径不存在，不能迁移。
- `目标目录已存在`：目标目录已有内容，应用不会覆盖，需要更换目标根目录或手动处理目标目录。
- `目标盘不存在或无法读取容量`：目标盘符不可用，检查磁盘是否挂载。
- `目标盘剩余空间不足`：目标盘可用空间小于源目录占用。
- `目标路径仍在系统盘`：可以继续作为警告展示，但不符合迁出系统盘目标。
- `completed-with-registry-failure`：文件移动有结果，但注册表写入失败，需要查看日志并谨慎处理。
- 失败项中出现文件占用或权限不足：对应文件未移动，应用不会强制处理。

### 注意事项

- 迁移只移动源目录的直接子项，不删除源目录本身。
- 目标路径已存在时不会覆盖。
- 跨盘迁移会使用受控分块复制、大小校验、删除源文件流程。
- 注册表更新后，资源管理器显示可能延迟，必要时重新登录或重启资源管理器。
- 当前未在本机真实用户目录上做自动测试，正式迁移前建议备份重要文件。

## 开发工具

### 功能

- 扫描开发工具、AI 工具、SDK、包管理器缓存和配置目录。
- 当前覆盖 Android、Cargo、Bun、通用缓存、Chromium snapshots、Claude、CodeBuddy、Apifox、AWS 等规则。
- 展示路径是否存在、占用大小、文件数量、风险等级和推荐动作。
- 对支持环境变量的工具生成迁移计划。
- 当前支持 `ANDROID_USER_HOME`、`CARGO_HOME`、`RUSTUP_HOME`。
- 执行环境变量迁移时只修改用户级环境变量并创建目标目录，不自动移动已有开发工具文件。
- 对支持 Junction 的规则生成迁移预案。
- 对可执行 Junction 预案执行真实迁移：移动目录到目标路径，并在原路径创建 Junction。

### 操作步骤

1. 进入“开发工具”页面。
2. 点击扫描按钮。
3. 等待 loading 文案结束。
4. 查看每个工具目录的占用和建议。

### 操作步骤：环境变量迁移

1. 进入“开发工具”页面。
2. 在“环境变量迁移目标”中输入目标根目录，例如 `D:\DevData`。
3. 在“规则文件工具项”中找到 Android 或 Rust Cargo。
4. 点击“环境变量计划”。
5. 在“环境变量迁移计划与结果”中确认变量原值、新值和目标路径。
6. 输入确认短语 `MIGRATE_DEVSPACE_ENV`。
7. 点击“执行”。
8. 在确认弹窗中再次确认变量变更。
9. 执行完成后到“日志”页面查看 `devspace-env-migration` 记录。

### 操作步骤：Junction 迁移预案

1. 进入“开发工具”页面。
2. 在目标根目录中输入计划迁出的目标位置，例如 `D:\DevData`。
3. 在“规则文件工具项”中找到支持 Junction 的工具。
4. 点击“Junction 预案”。
5. 在“Junction 迁移预案”中查看源路径、目标路径、占用大小、是否已有 Junction、阻塞项和警告。

### 操作步骤：执行 Junction 迁移

1. 进入“开发工具”页面。
2. 输入目标根目录，例如 `D:\DevData`。
3. 点击“Junction 预案”。
4. 确认预案状态为 `ready`，且没有阻塞项。
5. 输入确认短语 `CREATE_DEVSPACE_JUNCTION`。
6. 点击“执行 Junction”。
7. 在确认弹窗中再次核对源路径和目标路径。
8. 执行完成后到“日志”页面查看 `devspace-junction-migration` 记录。
9. 如需恢复，在“日志”页面点击“回滚 Junction”。

### 正确结果

- 扫描完成后显示各规则的路径、大小和状态。
- DevSpace 扫描期间会显示进度条和当前扫描工具名称。
- DevSpace 扫描期间可以点击“取消扫描”，任务会在下一次规则检查点停止。
- 高风险目录只展示，不提供自动处理入口。
- `.aws` 等凭据目录只读展示。
- Android 环境变量计划显示 `ANDROID_USER_HOME` 的原值和新值。
- Cargo 环境变量计划显示 `CARGO_HOME`、`RUSTUP_HOME` 的原值和新值。
- 输入完整 `MIGRATE_DEVSPACE_ENV` 后，执行按钮变为可用。
- 执行完成后状态为 `completed` 或 `completed-with-skips`。
- 日志页面出现 `devspace-env-migration` 类型记录，并在可回滚时显示“回滚环境变量”。
- Junction 预案没有阻塞项时显示 `ready`，可以输入确认短语后执行。
- Junction 预案能展示目标路径是否已存在、目标盘是否存在、目标空间是否足够。
- 执行 Junction 迁移后，目标路径包含原目录内容，原路径变为 Junction。
- 日志页面出现 `devspace-junction-migration` 类型记录，并在可回滚时显示“回滚 Junction”。

### 错误或异常结果

- 目录显示缺失：说明当前用户环境没有该工具目录。
- 访问失败或跳过项增加：说明存在权限限制或文件占用。
- 环境变量计划按钮不可用：说明该工具不支持环境变量迁移。
- `高风险目录不允许自动迁移`：该规则不能执行迁移。
- `该工具不支持环境变量迁移`：该规则没有环境变量迁移配置。
- 执行结果为 `completed-with-skips`：部分目标目录创建或环境变量写入失败，需要查看失败项。
- Junction 预案按钮不可用：说明该工具不支持 Junction，或该规则是高风险目录。
- Junction 预案出现 `源路径不存在`：当前电脑上没有该工具目录。
- Junction 预案出现 `目标路径已存在`：目标位置已有文件或目录，未来执行时不能覆盖。
- Junction 预案出现 `目标盘剩余空间不足`：目标盘容量不足，不适合迁移。
- Junction 预案出现 `该规则当前推荐只读`：该规则只允许评估，不允许真实执行。
- 执行 Junction 迁移失败：通常是文件占用、权限不足、目标路径冲突或 Junction 创建失败。
- 创建 Junction 失败后，应用会尝试把目标目录恢复回源路径；仍需查看失败项确认状态。

### 注意事项

- 当前不会清理开发工具目录。
- 只有可执行 Junction 预案会移动对应开发工具目录并创建 Junction。
- 环境变量变更后，新终端或重启后的开发工具更容易读取新值；已打开的终端通常不会自动更新。
- 删除 Junction 时必须通过日志回滚入口，不要手动删除目标目录。

## 分析

### 功能

- 展示系统盘概览。
- 展示规则路径扫描结果。
- 帮助判断空间占用来源。

### 操作步骤

1. 进入“分析”页面。
2. 点击扫描相关按钮。
3. 等待扫描完成。
4. 查看各路径的大小、文件数量、目录数量和跳过项。

### 正确结果

- 扫描期间有 loading 状态。
- Analysis 页面扫描期间会显示进度条和当前扫描规则名称。
- Analysis 页面扫描期间会显示当前扫描路径、文件数、目录数、跳过数和扫描字节数。
- Analysis 页面扫描期间可以点击“取消扫描”，任务会在下一次规则检查点停止。
- 扫描完成后显示每条规则的统计结果。
- 跳过项不会导致扫描整体失败。

### 错误或异常结果

- 路径不存在：说明规则路径在当前电脑上不存在。
- 权限不足：说明该目录受系统保护或当前用户无权访问。
- 扫描耗时较长：大目录或大量小文件会增加耗时。

## 日志与回滚

### 功能

- 查看清理、迁移、环境变量和 Junction 预案操作记录。
- 查看计划草稿、可回滚数量、异常任务数量和最近任务。
- 查看本地任务队列。
- 导出失败项 CSV。
- 打开失败项 CSV 导出目录。
- 回滚隔离清理。
- 回滚用户目录迁移。

### 操作步骤：回滚隔离清理

1. 进入“日志”页面。
2. 找到 `cleanup-quarantine` 类型记录。
3. 点击“回滚清理”。
4. 在确认弹窗中确认。
5. 查看恢复数量、跳过数量和失败项。

### 操作步骤：导出失败项

1. 进入“日志”页面。
2. 在“最近任务”中找到失败项数量大于 0 的记录。
3. 点击“导出失败项”。
4. 查看“导出结果”中的文件路径。
5. 点击“打开导出目录”可以在资源管理器中打开导出目录。
6. 导出的 CSV 位于 `.clearc\exports`。

### 操作步骤：查看本地任务队列

1. 进入“日志”页面。
2. 查看“本地任务队列”。
3. 点击“刷新任务状态”读取最新本地任务记录。
4. 点击“清理已完成任务”只保留运行中任务。

### 操作步骤：回滚用户目录迁移

1. 进入“日志”页面。
2. 找到 `user-folder-migration` 类型记录。
3. 点击“回滚迁移”。
4. 在确认弹窗中确认。
5. 查看恢复数量、跳过数量、注册表恢复状态和失败项。

### 操作步骤：回滚开发工具环境变量迁移

1. 进入“日志”页面。
2. 找到 `devspace-env-migration` 类型记录。
3. 点击“回滚环境变量”。
4. 在确认弹窗中确认。
5. 查看恢复数量、跳过数量和失败项。

### 操作步骤：回滚开发工具 Junction 迁移

1. 进入“日志”页面。
2. 找到 `devspace-junction-migration` 类型记录。
3. 点击“回滚 Junction”。
4. 在确认弹窗中确认。
5. 查看恢复数量、跳过数量和失败项。

### 正确结果

- 可回滚记录显示回滚按钮。
- 回滚期间显示 loading 文案。
- 回滚成功后写入新的不可回滚日志。
- 同一个迁移或隔离清理不能重复回滚。
- 开发工具环境变量回滚会恢复原值；原值不存在时删除该变量。
- 开发工具 Junction 回滚会先删除原路径 Junction，再把目标目录恢复到原路径。
- 用户目录迁移回滚期间会显示进度条和当前处理内容。
- 用户目录迁移回滚期间会显示已处理项、已恢复项、跳过项和失败项。
- 用户目录迁移回滚跨盘复制期间会显示当前文件复制字节数。
- 用户目录迁移回滚期间可以点击“取消回滚”，任务会在下一次文件恢复检查点停止。
- 有失败项的任务显示“导出失败项”按钮。
- 导出成功后显示导出条数和 CSV 文件路径。
- “打开导出目录”只打开 `.clearc\exports`，不会打开任意用户输入路径。
- 离开日志页再返回，仍然可以看到最近任务结果。
- 清理、迁移、开发工具、分析页面的长任务会写入本地任务队列。
- 已完成、失败、取消和运行中任务会显示不同状态。

### 错误或异常结果

- `operation is not rollbackable`：该记录不是可回滚操作。
- `migration already has a rollback record`：该迁移已经回滚过，不能重复回滚。
- 原路径已存在：应用不会覆盖已有文件，失败项会记录。
- 目标文件缺失：待恢复文件不存在，失败项会记录。
- `rolled-back-with-registry-failure`：文件回滚有结果，但注册表恢复失败，需要手动检查 Windows 默认目录配置。
- `devspace env migration already has a rollback record`：该开发工具环境变量迁移已经回滚过。
- 环境变量删除或恢复失败：需要查看失败项，并手动检查用户级环境变量。
- `devspace junction migration already has a rollback record`：该 Junction 迁移已经回滚过。
- Junction 回滚失败：通常是原路径不是 Junction、目标目录缺失、文件占用或权限不足。
- `operation has no failure items to export`：该任务没有失败项，不能导出。
- 导出失败：通常是 `.clearc\exports` 无法创建或文件写入失败。
- 打开导出目录失败：通常是系统资源管理器不可用或应用数据目录无法创建。
- 本地任务显示 running 但实际已结束：说明页面关闭或刷新时任务状态没有机会写回，可清理已完成任务或重新执行操作。

## 白屏或页面异常

### 功能

- 应用已加入前端错误兜底。
- 页面渲染异常时会显示“页面渲染异常”面板，而不是整页白屏。

### 正确结果

- 普通操作不会出现白屏。
- 如果出现错误面板，可以看到错误信息，并点击“重新显示页面”尝试恢复。

### 错误或异常结果

- 如果仍然整页白屏，说明异常发生在错误兜底之前或 WebView 本身崩溃。
- 此时需要记录操作步骤、最后点击的按钮、输入内容，并查看开发日志。

## 判断操作是否正确

| 操作类型 | 正确表现 | 错误或需要关注 |
|---|---|---|
| 扫描 | 有 loading，完成后显示统计结果 | 长时间无响应、无错误提示、页面白屏 |
| 扫描进度 | Analysis 页面显示进度条和当前规则 | 扫描中没有任何进度反馈 |
| 递归扫描进度 | Analysis/清理预览显示当前路径、文件数、目录数和字节数 | 只能看到规则级进度，看不到目录内部变化 |
| DevSpace 扫描进度 | 开发工具页面显示进度条和当前工具 | 扫描开发工具时没有当前工具提示 |
| 清理进度 | 清理页面显示预览或隔离进度 | 清理按钮 loading 但没有进度条 |
| 隔离复制进度 | 隔离清理复制 fallback 显示当前文件字节数 | 大目录复制时没有内部进度 |
| 迁移进度 | 迁移页面显示预检查、计划、执行进度 | 迁移中没有任何进度反馈 |
| 迁移回滚进度 | 日志页显示回滚进度 | 回滚中没有任何进度反馈 |
| 条目级统计 | 清理/迁移显示已处理、跳过、失败和字节数 | 只有进度条，无法判断处理了多少项 |
| 字节级迁移进度 | 迁移跨盘复制显示当前文件已复制字节和总字节 | 大文件复制时只有条目进度 |
| 任务取消 | 点击取消后任务在下一次检查点停止，并显示 cancelled 状态 | 点击取消后仍持续处理大量项目、没有取消状态 |
| 任务状态 | 最近任务显示状态和失败项数量 | 任务完成后日志页没有记录 |
| 本地任务队列 | 显示 running/completed/failed 状态 | 离开页面后完全丢失最近任务 |
| 预检查 | 显示通过或明确阻塞原因 | 阻塞原因为空但按钮不可用 |
| 生成计划 | 显示源路径、目标路径、ready/blocked | 计划路径不符合预期 |
| 执行迁移 | 要求确认短语和弹窗确认 | 未确认就执行、目标路径已存在仍执行 |
| 环境变量迁移 | 要求确认短语，日志记录原值和新值 | 高风险目录可执行、未记录原值 |
| Junction 预案 | 显示 ready/blocked 和阻塞原因 | 目标冲突仍可执行 |
| Junction 执行 | 要求确认短语，日志记录移动项，可回滚 | 未确认就创建 Junction、只读规则可执行 |
| 清理 | 只隔离移动，不永久删除 | 删除了非临时目录或系统目录 |
| 回滚 | 写入新日志，重复回滚被拒绝 | 覆盖已有文件、重复回滚成功 |
| 失败项导出 | 生成 CSV 并显示路径 | 无失败项却可导出、CSV 缺少路径或原因 |
| 打开导出目录 | 打开 `.clearc\exports` | 打开了非应用导出目录或任意用户输入路径 |
